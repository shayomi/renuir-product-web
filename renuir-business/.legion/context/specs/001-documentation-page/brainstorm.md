# Brainstorm: Documentation Page Setup

Create a first-class docs hub for Renuir partners and developers.

## Problem Space
- Business partners have no authoritative reference when onboarding to the API or dashboard exports.
- Growth needs a linkable destination before campaigns start driving traffic to “Docs”.
- Success/support wastes time re-sending PDF playbooks that are already out of date.

## Signals & Insights
- Two hotel pilots asked for “a single URL with auth, rate limits, errors, everything”.
- 28% of marketing site traffic clicks “Developers” in nav but lands on a placeholder hero.
- Support macros referencing Google Docs have 48 hour update lag.

## Idea Seeds
- Opinionated hero with CTA buttons (Get API Key, Talk to Solutions) plus highlight cards for SDKs.
- Quickstart timeline (Auth, Create Lost Item, Webhook) with copy/paste code snippets.
- Accordion-based API reference fed by static JSON for v1 endpoints.
- Downloadable assets (Postman collection, schema diagram) to reduce repeated requests.

## Risks & Unknowns
- Content freshness: who owns updating examples as API evolves?
- Depth vs discoverability: avoid overwhelming individuals who just want to read FAQs.
- Future localization needs—structure should keep translation paths open.
