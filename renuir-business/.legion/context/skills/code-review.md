# Skill: Code Review
## When to use
Use this skill before merging changes to ensure quality and convention alignment.

## Steps (numbered)
1. Check server vs client component boundaries for misuse in Next.js App Router.
2. Verify TypeScript types are present and avoid unsafe implicit any patterns.
3. Ensure Tailwind usage is consistent with project styling conventions.
4. Validate accessibility basics (semantic markup, labels, keyboard support, contrast).
5. Review test coverage for changed logic and critical flows.
6. Call out risks, required fixes, and optional follow-up improvements.

## Output
A review report highlighting correctness, architecture alignment, accessibility, and test coverage gaps with clear remediation guidance.
