# Skill: API Endpoint
## When to use
Use this skill when designing or implementing a new REST API route for Renuir.

## Steps (numbered)
1. Define the REST route resource, method, and versioned path with clear naming.
2. Specify request and response contracts, including required and optional fields.
3. Implement input validation with explicit schema/type checks.
4. Add structured error handling with stable status codes and error payloads.
5. Include authentication/authorization checks where applicable.
6. Update `.legion/context/decisions.md` if a new API pattern or convention is introduced.

## Output
A consistent REST endpoint design and implementation with validation, predictable errors, and recorded architectural decisions when new patterns appear.
